## Availability

The dashboard is available in the `dashboard-*` Docker Compose profiles.

Example:

```bash
COMPOSE_PROFILES= docker compose --profile dashboard-postgres-ghcr up -d
```

Then open:

- `http://localhost:8081/decision-engine/`

The dashboard is served by Nginx from the built assets in `website/dist`.
This docs page intentionally lives at `/dashboard-guide`; `/decision-engine/` is reserved for the React app in production-style hosting.

## What It Includes

Based on the React routes in `website/src/App.tsx`, the dashboard exposes:

- `/decision-engine/`
- `/decision-engine/login`
- `/decision-engine/signup`
- `/decision-engine/onboarding`
- `/decision-engine/routing`
- `/decision-engine/routing/sr`
- `/decision-engine/routing/rules`
- `/decision-engine/routing/volume`
- `/decision-engine/routing/debit`
- `/decision-engine/decisions`
- `/decision-engine/analytics`
- `/decision-engine/audit`

For source-run Vite development, the same React routes are mounted at root, for example `http://localhost:5173/routing/rules`.
Production builds default to `/decision-engine/`. To override the hosted prefix, run `VITE_DASHBOARD_BASE_PATH=/your-prefix/ npm run build` from `website/`.
Local dev API calls use `/decision-engine-api/*`, which Vite proxies to `http://localhost:8080`.
Production builds default to `https://app.hyperswitch.io/decision-engine/api/*`.
Override it with `VITE_API_BASE_PATH=https://your-host/decision-engine/api`.

## Docs And API In The Same Profile

The same `dashboard-*` profiles also expose:

- Mintlify docs at `http://localhost:8081/introduction`
- API examples at `http://localhost:8081/api-refs/api-ref`
- the API at `http://localhost:8080`

Proxy behavior is defined in `nginx/nginx.conf`.
