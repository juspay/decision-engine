# Delete Rule Config

## Use case

Deletes a service-level success-rate or elimination configuration for a merchant.

## Authentication

Protected. Send either `Authorization: Bearer <jwt_token>` or `x-api-key: <api_key>`.

For local development, start with:

```bash
export BASE_URL=http://localhost:8080
export AUTH_HEADER="Authorization: Bearer <jwt_token>"
```

## Request

- Method and path: `POST /rule/delete`
- Parameters: none.
- Body: JSON body with `merchant_id` and `config.type` to delete.

## Example

### Delete success-rate config

```bash
curl --location "$BASE_URL/rule/delete" \
  --header "$AUTH_HEADER" \
  --header "Content-Type: application/json" \
  --data '{
  "merchant_id": "merchant_demo",
  "config": { "type": "successRate" }
}'
```

### Delete elimination config

```bash
curl --location "$BASE_URL/rule/delete" \
  --header "$AUTH_HEADER" \
  --header "Content-Type: application/json" \
  --data '{
  "merchant_id": "merchant_demo",
  "config": { "type": "elimination" }
}'
```

## Response

```json
{
  "merchant_id": "merchant_demo",
  "config_type": "successRate",
  "deleted": true
}
```

## Notes

- Deleting config makes the dashboard show that surface as not configured.
- Routing may fall back to defaults if config is absent.
